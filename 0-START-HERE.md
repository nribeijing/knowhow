## 🎯 KnowHow 是什么？

KnowHow 是团队的**知识管理平台**，包含三类知识：

```
knowledge   →  必须知道的标准化知识 (规则/标准/原则)
experience  →  可以学习的经验性知识 (案例/模板/指南)
skills      →  可以执行的通用技能 (流程/步骤/动作)
```

---

## 📚 目录结构详解

### knowledge/ - 标准化知识

**特点**: 静态、通用、必须遵守

| 目录 | 内容 | 示例 |
|------|------|------|
| **wiki/** | 通用术语、基础概念 | 待添加 |
| **principles/** | 设计原则 | DRY, SOLID, RESTful |
| **standards/** | 编码/交付标准 | PEP8, ESLint |
| **best-practices/** | 最佳实践 | API 设计最佳实践 |
| **rules/** | 硬约束、禁令 | API 斜杠规则 |

---

### experience/ - 经验性知识

**特点**: 动态、场景化、参考学习

| 目录 | 内容 | 时间维度 |
|------|------|----------|
| **case-studies/** | 案例复盘 | 过去 (已发生) |
| **workflow/** | 工作流程 | 现在 (执行中) |
| **guides/** | 指导手册 | 未来 (预防) |
| **templates/** | 模板 | 随时使用 |
| **checklists/** | 检查清单 | 随时使用 |
| **agents/** | AI Agent 配置 | 随时使用 |

---

### skills/ - 通用技能

**特点**: 流程化、可执行、可组合

| 目录 | 内容 | 使用场景 |
|------|------|----------|
| **code/** | 代码技能 | 写代码/调试/重构 |
| **review/** | 评审技能 | 代码审查 |
| **design/** | 设计技能 | 架构/接口设计 |
| **deploy/** | 部署技能 | 部署/运维 |

## 💡 使用技巧

### 1. 知道类型就能找到

```
要找规则？ → knowledge/rules/
要学案例？ → experience/case-studies/
要用模板？ → experience/templates/
要执行技能？ → skills/
```

### 2. 按工作流程使用

```
项目开始 → experience/workflow/
写代码   → skills/code/ + experience/templates/
代码审查 → skills/review/ + experience/checklists/
部署上线 → skills/deploy/ + experience/checklists/
问题复盘 → experience/case-studies/
```

### 3. 新人学习路径

```
第 1 天：沟通原则 + 任务描述模板
第 1 周：AI 辅助编程 + 检查清单
第 1 月：技术经验 + 代码技能
第 3 月：设计技能 + 案例学习
```
