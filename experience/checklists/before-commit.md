# 代码提交前检查清单

> 确保每次提交都符合质量标准

**适用场景**: 代码提交前 AI 自查 + 人类审查  
**执行者**: AI (自查) + 人类 (审查)  
**耗时**: 5-10 分钟

---

## ✅ 功能正确性

### 测试覆盖
- [ ] 所有单元测试通过
- [ ] 集成测试通过 (如适用)
- [ ] 边界条件已测试
- [ ] 异常路径已测试

### 功能验证
- [ ] 需求全部实现
- [ ] 业务逻辑正确
- [ ] 错误处理完整
- [ ] 日志记录适当

### 回归测试
- [ ] 现有功能未受影响
- [ ] 相关测试已更新
- [ ] 性能无明显下降

## ✅ 代码质量

### 代码规范
- [ ] 遵循项目代码规范
- [ ] 无 lint 错误
- [ ] 无 format 警告
- [ ] 命名清晰易懂

### 代码结构
- [ ] 无重复代码 (DRY)
- [ ] 函数职责单一 (SRP)
- [ ] 模块依赖合理
- [ ] 路由顺序正确 (具体在前，参数在后)

### 注释文档
- [ ] 复杂逻辑有注释
- [ ] 公共 API 有文档
- [ ] 参数说明完整
- [ ] 返回值说明清晰

## ✅ 安全性

### 输入验证
- [ ] 所有输入已验证
- [ ] 类型检查完整
- [ ] 长度限制适当
- [ ] 格式验证正确

### 权限控制

## ✅ 测试质量

### Mock 验证
- [ ] Mock 正确 (异步方法用函数包装)
- [ ] Mock 范围适当 (不过度 Mock)
- [ ] 外部依赖已 Mock (ES/Redis/第三方 API)
- [ ] Mock 数据完整

### 测试隔离
- [ ] 无全局状态污染
- [ ] 使用 autouse fixture 清理
- [ ] 测试间无依赖
- [ ] 测试容器独立

### 响应验证
- [ ] 返回数据包含所有必填字段
- [ ] 类型正确
- [ ] 无 None 值 (除非 Optional)
- [ ] Pydantic 验证通过

## 📋 自查流程

### AI 自查 (必须)

```markdown
1. 运行所有测试
   命令：uv run pytest tests/ -v

2. 运行代码检查
   命令：uv run ruff check .
   命令：uv run black --check .

3. 按本清单逐项自查
   输出：自查报告 (每项的结果)

4. 修复发现的问题
   重新运行测试验证

5. 生成提交信息
   格式：type: description
```

### 人类审查 (关键节点)

```markdown
1. 审查 AI 自查报告
   重点：是否有遗漏项

2. 抽查核心代码
   重点：业务逻辑、安全性

3. 确认测试覆盖
   重点：边界条件、异常路径

4. 批准或要求修改
   决定：批准提交 / 需要修改
```

## 🚨 红线项目 (必须人类审查)

以下项目 **必须** 人类审查，AI 不得自行决定：

- 🔴 核心业务逻辑变更
- 🔴 安全性相关代码
- 🔴 数据库结构变更
- 🔴 外部接口变更
- 🔴 性能关键路径
- 🔴 权限控制变更
