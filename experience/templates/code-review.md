## 🔍 审查清单

### 1. 功能正确性

- [ ] 需求是否完整实现
- [ ] 业务逻辑是否正确
- [ ] 边界条件是否处理
- [ ] 异常流程是否考虑
- [ ] 所有测试通过

### 2. 代码质量

- [ ] 遵循代码规范
- [ ] 无重复代码 (DRY)
- [ ] 函数职责单一 (SRP)
- [ ] 命名清晰易懂
- [ ] 注释适当

### 3. 安全性

- [ ] 输入验证完整
- [ ] 权限检查到位
- [ ] 无 SQL/NoSQL 注入风险
- [ ] 无敏感信息泄露
- [ ] 依赖安全

### 4. 性能

- [ ] 无明显性能瓶颈
- [ ] 数据库查询优化
- [ ] 无 N+1 查询
- [ ] 缓存使用适当
- [ ] 资源释放及时

### 5. 测试

- [ ] 测试覆盖充分 (>85%)
- [ ] 测试独立
- [ ] Mock 正确
- [ ] 测试数据合理

### 6. 文档

- [ ] README 已更新
- [ ] API 文档已更新
- [ ] 提交信息规范
- [ ] CHANGELOG 已更新
