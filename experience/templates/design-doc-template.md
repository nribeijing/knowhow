# 设计文档模板

> 技术方案标准化模板 · AI 辅助编写指南

**适用场景**: 核心功能开发/架构变更/新技术引入  
**详细程度**: 根据功能复杂度选择 L1/L2/L3  
**AI 职责**: 按模板编写 + 按清单自查

---

## 📋 模板选择

### L1 - 基础级 (小功能/Bug 修复)

**适用场景**:
- Bug 修复
- 小功能增强
- 配置变更

**内容结构**:
```markdown
## 功能描述
[1-2 段话描述要做什么]

## 技术方案
[简单描述实现方案]

## 影响范围
[影响的模块/文件]
```

**时间投入**: 0.5-1 小时

---

### L2 - 标准级 (中等功能) ⭐ 推荐使用

**适用场景**:
- 新功能开发
- 模块重构
- 技术升级

**内容结构**:
```markdown
# Design Doc: [功能名称]

## 1. 问题陈述
[为什么需要做这个功能]

## 2. 目标
[3-5 个可衡量的目标]

## 3. 技术方案
- 架构图
- 数据模型
- API 设计

## 4. 测试策略
- 单元测试
- 集成测试

## 5. 风险评估
- 技术风险
- 缓解措施
```

**时间投入**: 2-4 小时

## 1. 概述 (Overview)

### 1.1 问题陈述

**用一句话说清楚**:
```

### 1.2 目标 (Goals)

**SMART 原则**:
- ✅ Specific (具体)
- ✅ Measurable (可衡量)
- ✅ Achievable (可实现)
- ✅ Relevant (相关)
- ✅ Time-bound (有时限)

```
1. [示例] 用户可以撤回状态为"pending_node1"的预约
2. [示例] 撤回操作响应时间 < 500ms
3. [示例] 撤回后预约状态变为"withdrawn"
```

### 1.3 非目标 (Non-Goals)

**明确不做什么**:
```

2. [示例] 不提供批量撤回功能

```

## 7. 监控告警 (Monitoring & Alerting)

### 7.1 关键指标 (SLO/SLI)

| 指标 | 目标值 | 测量方法 |
|------|--------|----------|
| 可用性 | > 99.9% | HTTP 200 成功率 |
| 延迟 P50 | < 200ms | 响应时间中位数 |
| 延迟 P99 | < 500ms | 99 百分位响应时间 |
| 错误率 | < 0.1% | 5xx 响应比例 |

### 7.2 告警级别

| 级别 | 条件 | 响应时间 | 通知方式 |
|------|------|----------|----------|
| P0 | 服务不可用 | 5 分钟 | 电话 + 短信 |
| P1 | 错误率 > 1% | 30 分钟 | 短信 + 邮件 |
| P2 | 延迟 P99 > 1s | 24 小时 | 邮件 |

### 7.3 仪表盘

**监控面板**:
- 业务指标 (撤回次数/撤回原因分布)
- 技术指标 (API 响应时间/错误率)
- 资源指标 (CPU/内存/数据库连接)

## ✅ 设计文档质量检查清单

### 内容完整性

- [ ] 问题陈述清晰 (1 句话说清楚)
- [ ] 目标可衡量 (SMART 原则)
- [ ] 架构图完整 (至少 1 个)
- [ ] 数据模型详细 (ER 图或 Schema)
- [ ] API 设计完整 (请求/响应示例)
- [ ] 替代方案分析 (至少 2 个方案)
- [ ] 测试策略详细 (覆盖率目标)
- [ ] 监控指标明确 (SLO/SLI)
- [ ] 风险评估完整 (概率 + 影响)
- [ ] 实施计划可行 (时间 + 交付物)

### 技术深度

- [ ] 技术选型有依据 (对比分析)
- [ ] 性能考虑充分 (QPS/延迟)
- [ ] 扩展性设计合理 (水平/垂直)
- [ ] 安全性考虑周全 (认证/授权/加密)
- [ ] 容错设计完善 (重试/降级/熔断)

### 文档质量

- [ ] 结构清晰 (有目录/标题)
- [ ] 语言简洁 (无 jargon)
- [ ] 图表丰富 (架构图/流程图/ER 图)
- [ ] 示例完整 (代码/配置/请求)
- [ ] 链接有效 (相关文档)
- [ ] 版本管理 (日期/作者/版本)

## 🤖 AI 编写指南

### AI 在这个阶段的任务

1. **理解需求**
   - 阅读需求文档
   - 澄清模糊点
   - 确认验收标准

2. **编写 Design Doc**
   - 按模板结构编写
   - 提供替代方案分析
   - 评估风险和成本

3. **按清单自查**
   - 内容完整性
   - 技术深度
   - 文档质量

4. **准备评审**
   - 提前 24 小时发送给评审者
   - 准备讲解材料
   - 收集反馈意见
