## 使用示例

### 示例 1：Bug 修复

```markdown
## 任务类型
- [x] Bug 修复

## 任务标题
修复"我的预约"列表返回空数据的问题

## 背景
用户报告访问 /reservations/mine 返回空数组，但数据库中确实有预约数据。

## 任务描述
1. 检查后端 API 是否正常返回数据
2. 检查 Elasticsearch 查询是否正确
3. 检查字段类型和查询条件
4. 修复问题并测试

## 验收标准
- [ ] 用 user004@test.com 登录能看到预约列表
- [ ] API 返回正确的数据
- [ ] 其他用户的预约列表也正常

## 相关文件
- backend/app/services/reservation_service.py
- backend/app/api/reservations.py

## 参考资源
- [Elasticsearch 查询文档](https://www.elastic.co/guide/)

## 时间要求
- 期望完成时间：今天
- 优先级：高
```

### 示例 2：功能需求

```markdown
## 任务类型
- [x] 新功能

## 功能概述

**用户故事**：
作为一般用户，我想要查看我的预约列表，以便了解我的预约状态。

**功能描述**：
在"我的预约"页面展示当前用户的所有预约记录，支持分页和筛选。

**功能要求**：
- [ ] 展示最近 90 天的预约
- [ ] 支持分页（每页 10 条）
- [ ] 按状态筛选（待审批/已通过/已拒绝）
- [ ] 点击详情跳转到预约详情页

**验收标准**：
- [ ] 登录用户能看到自己的预约
- [ ] 未登录用户跳转到登录页
- [ ] 分页功能正常
- [ ] 筛选功能正常

**UI/UX 参考**：
（设计稿链接）

**技术考虑**：
- 前端：Vue3 + Axios
- 后端：GET /api/v1/reservations/mine
- 数据库：从 Elasticsearch 查询
```
